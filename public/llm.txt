# GEO Engine - Generative Engine Optimization for Shopify

GEO Engine is the operating system for AI Commerce. It evaluates, optimizes, and prepares Shopify store product listings, images, variants, and policies for discovery and purchase by AI agents, LLM search engines (Google Gemini, ChatGPT, Perplexity, Claude), and custom shopping crawlers.

---

## 🏗️ Architecture & Core Components

GEO Engine is built on top of React Router flat routes (Remix template framework) using Shopify App Bridge, SQLite database via Prisma, and background worker queues.

## 🗄️ Database Schema (Prisma Models)

The SQLite database schema details (`schema.prisma`):

*   **StoreProfile**: Holds store identity, scanning statuses (`IDLE`, `SCANNING`), active subscription plans (`FREE`, `STARTER`, `GROWTH`, `UNLIMITED`), and daily limits.
*   **Scan**: Logs historical health runs, storing overall score averages and counts of critical, warning, and info issues.
*   **ProductScan**: Logs telemetry grades for individual products.
*   **ScanIssue**: Identifies visibility issues mapping to rule categories (`TITLE`, `DESCRIPTION`, `IMAGES`, `VARIANTS`, `METADATA`, `POLICIES`).
*   **AISuggestion**: Holds proposed adjustments for Title, Description, and Alt text generated by OpenAI or local heuristics.
*   **ScanHistory**: Records daily aggregated readiness score data points for sparkline visualization.
*   **Competitor & CompetitorScan**: Collects competitor metrics.

## 🔍 Scan Rules & Scoring Metrics

Taramalar deterministic ve heuristic kurallar çerçevesinde yapılır:

1.  **Title Evaluation**:
    *   *Warning*: Titles under 20 or over 70 characters (penalizes visibility score by 20%).
    *   *Warning*: Presence of sales-buzzwords like "Bestseller", "Guaranteed", "Cheap", "Promo" (penalizes by 15%).
2.  **Description Evaluation**:
    *   *Critical*: Empty description (penalizes score by 50%).
    *   *Warning*: Lacks HTML structures (lists, bullet points, header tags) which decreases rich text readability for scrapers (penalizes score by 15%).
3.  **Image Evaluation**:
    *   *Critical*: Visual media with blank or default image filenames without custom Alt descriptive text (penalizes by 30%).
4.  **Variant Evaluation**:
    *   *Warning*: Missing SKU codes (penalizes by 25%).
    *   *Warning*: Missing Barcode/EAN/GTIN values (penalizes by 20%).
    *   *Warning*: Missing physical weight or zero price attributes.
5.  **Policy Schema Evaluation**:
    *   *Warning*: Absence of clear Refund Policy, Shipping Policy, or Terms of Service documents.

## 🧮 ROI Priority Calculations

All detected scan issues are prioritized using the following ROI Priority Matrix:

$$\text{ROI Score} = \frac{\text{Severity Impact Score (1 to 10)}}{\text{Difficulty Effort Score (1 to 5)}}$$

*   **Impact Score**: `CRITICAL` = 10, `WARNING` = 5, `INFO` = 2.
*   **Effort Score**: `EASY` = 1, `MEDIUM` = 2, `HARD` = 4.
*   Issues are ranked highest ROI to lowest. This helps merchants fix high-impact, easy-to-solve issues first to maximize AI visibility with minimal labor.

## 🤖 AI Enrichment Pipeline

Suggestions are built using an automated hybrid flow:
1.  **GPT-4o-mini structured JSON outputs**: Sends existing product parameters to OpenAI chat completion pipeline. Returns optimized, structured fields.
2.  **Deterministic Rules Fallback**: If `OPENAI_API_KEY` is not configured in `.env`, the system automatically falls back to an offline heuristic engine to generate suggestions based on category definitions, ensuring seamless local operations.

## 🚦 Application Routes & UX Flow

*   **Dashboard (`/app`)**: Telemetry cockpit. Displays Circular dial charts, categorical progress bars, policy indicators, prioritized recommendations table, and historic trends.
*   **Catalog Directory (`/app/products`)**: Filterable data grid of all catalog items, showing individual product health scores.
*   **Suggestion Review Card (`/app/product/$id`)**: Visual comparison editor showing the original content side-by-side with AI recommendation. Allows editing, dismissing, or syncing back to Shopify.
*   **Billing Page (`/app/billing`)**: Pricing package matrix with tier upgrades.
